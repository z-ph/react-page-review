# Proposal: 修复评审弹窗滚动与焦点问题

## Change ID

`fix-modal-scroll-focus`

## Why

当前「添加评审意见」弹窗（Ant Design Modal）打开时存在两个可用性问题：

1. **页面仍可滚动**：弹窗显示后，鼠标滚轮或触控板滚动会作用于弹窗背后的页面，而不是弹窗内容。尤其在长页面中，用户容易误操作导致页面滚动、选中元素偏移，影响评审体验。
2. **焦点未进入弹窗**：打开弹窗后键盘焦点仍留在原页面元素上，按 Tab 会循环页面元素而不是在弹窗内跳转，Esc 关闭弹窗后焦点也不会回到触发按钮。

这两个问题降低了评审流程的可访问性和稳定性。

## What Changes

1. **弹窗打开时锁定页面滚动**
   - 当 `formVisible`、`treeVisible`、`listVisible` 或 `confirm` 任一弹窗/抽屉打开时，给 `document.body` 添加 `overflow: hidden`（并保留原始值以便恢复）。
   - 关闭所有弹窗/抽屉后恢复原始 `overflow`。

2. **焦点移入弹窗**
   - 评审弹窗打开后，将焦点移动到弹窗内第一个可聚焦元素（标题输入框）。
   - 确认弹窗打开后，将焦点移动到「确定」按钮。

3. **关闭后恢复焦点**
   - 关闭评审弹窗或确认弹窗时，将焦点恢复到打开前的触发元素（工具栏「评审」按钮或操作按钮）。

4. **新增 e2e 断言**
   - 打开评审弹窗后断言 `body` 的 `overflow` 为 `hidden`。
   - 断言弹窗内标题输入框获得焦点。
   - 关闭弹窗后断言焦点回到触发按钮。

## Impact

- **代码**：`src/ReviewTool.jsx`、`src/style.css`、`test-react-e2e.cjs`
- **行为**：弹窗打开期间页面无法滚动；焦点进入弹窗；关闭后焦点还原
- **无破坏性变更**：不修改评审记录结构、截图逻辑或公开 API

## 验证

- 在 `react-test-app` 端到端用例中新增弹窗焦点/滚动场景：
  - 选中元素并打开评审弹窗，断言 `document.body.style.overflow === 'hidden'`
  - 断言弹窗内标题输入框为 `document.activeElement`
  - 关闭弹窗后断言工具栏「评审」按钮重新获得焦点
  - 既有 40 项 e2e 全部保持通过
