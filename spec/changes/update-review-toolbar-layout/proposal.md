# Proposal: 调整评审工具栏布局

## Change ID

`update-review-toolbar-layout`

## Why

当前评审工具栏存在两个交互不够直观的问题：

1. **「取消选择」藏在「更多」下拉菜单中**：用户选中元素后若要清空选择，需要点击「更多」才能看到「取消选择」，操作路径较长，不符合常用功能就近放置的原则。
2. **模式切换使用 Radio Tabs**：「选中元素」与「框定视图」以 Radio Button 组形式平铺展示，在工具栏有限空间内显得拥挤，且视觉上更像筛选器而非模式切换。

将「取消选择」外露到工具栏，并把模式切换改为下拉选择，可以让工具栏更简洁、核心操作更易触达。

## What Changes

1. **将「取消选择」移出下拉菜单**
   - 在工具栏右侧「评审」与「退出评审」按钮附近新增一个「取消选择」按钮（文字或图标按钮）。
   - 仅在有选中元素/框选时可用（`disabled={selectedCount === 0}`）。
   - 从「更多」下拉菜单中移除「取消选择」项。

2. **将 Radio Tabs 改为下拉选择**
   - 将工具栏左侧的 `Radio.Group`（「选中元素」/「框定视图」）替换为原生 `<select>` 下拉选择。
   - 下拉选项显示当前模式：
     - `element` → 「选中元素」
     - `viewport` → 「框定视图」
   - 下拉宽度保持紧凑，默认展示当前选中的模式名称。
   - 增加 `data-testid="rpr-mode-select"` 便于 e2e 定位。

3. **样式适配**
   - 调整工具栏左右两侧间距，确保新增按钮不溢出。
   - 下拉选择框与周围按钮在视觉高度、圆角上保持一致。

4. **新增 e2e 断言**
   - 断言工具栏上存在「取消选择」按钮。
   - 断言未选中元素时「取消选择」按钮禁用，选中后启用。
   - 断言模式切换下拉存在且可在两个选项间切换。
   - 断言「更多」下拉中不再包含「取消选择」。

## Impact

- **代码**：`src/ReviewTool.jsx`、`src/style.css`、`test-react-e2e.cjs`
- **UI**：工具栏布局变化；模式切换由 Tabs 变为下拉；「取消选择」外露
- **行为**：功能不变，仅交互入口调整
- **无破坏性变更**：公开 API、评审记录结构、截图逻辑均不受影响
